题目描述

如果一个正整数的各个数位上的数都为 $6$ 或者 $8$，则称吉利数，求区间 $[a, b]$ 内有多少数是任意一个吉利数的倍数。

输入格式

输入共一行，包含两个正整数 $a, b$ ($a \leq b \leq 10^9$)。

输出格式

输出一行一个整数，表示区间 $[a, b]$ 内满足条件的数的个数。

题解

首先，这没有什么比较好的数学方法 (比如筛法等等)，考虑优化暴力，由于 $9$ 位及以内的吉利数不超过 $1 + 2 + \cdots + 2^9 = 1023$ 个，然后可以把互相整除的吉利数中大的扔掉 (比如吉利数 $a, b$ 有 $a \mid b$，则 $b$ 的倍数一定是 $a$ 的倍数)。

那么只会剩下 $472$ 个吉利数。

考虑一般的暴力判断，那么复杂度就为 $O \left( 472(b-a) \right)$，显然要 TLE (不过可以骗 $50$ 分？)。

首先，由于所有的奇数一定不满足条件，且一个偶数满足条件当且仅当它的一半可以被 "亚吉利数" 整除，其中，"亚吉利数" 是只含 $3$ 或 $4$ 的数。

那么一共有 $5 \times 10^8$ 个数，可以考虑用分段打表进行优化。考虑将它分成 $50000$ 个长度为 $10000$ 的区间，预处(da)理(biao)出每个区间中满足条件的数的个数，然后分个块就可以了。

ps: 可以发现，每个区间中满足条件的数的个数都是比较相近的，极差不超过 $31$，因此可以进行压缩打表，用一个字节表示一个区间，在程序头部解密就可以了。

int val[50001];

int a, b, l, r, i, sum = 0;

void For(int u, int v){
    for(int j; u < v; sum += j < 472, ++u)
        for(j = 0; j < 472; ++j)
            if(!(u % good[j])) break;
}

int main(){
    for(val[0] = i = 0; i < 50000; ++i) val[i + 1] = val[i] + 5196 + table[i];
    scanf("%d%d", &a, &b); ++a >>= 1; ++(b >>= 1);
    l = (a + 9999) / 10000; r = b / 10000;
    if(l <= r){
        sum = val[r] - val[l];
        For(a, l * 10000);
        For(r * 10000, b);
    }else
        For(a, b);
    printf("%d\n", sum);
    return 0;
}
